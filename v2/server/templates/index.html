  • Project Description
    • The Green Smart Home project is a smart home management system with a microservices architecture. It uses Docker for deployment, Hazelcast for real-time distributed data management, and the MVC pattern for the web service. Data from LoRaWAN sensors are collected through a MQTT broker integrated into the "The Things Network Sciencetack". This data are then used as the basis for controlling physical devices based on the values.
    A single AI client, implemented in Java, manages multiple virtual workers (agents) to complete tasks. It uses planning techniques to prevent delays and conflicts when agents operate simultaneously. Particularly in loosely connected tasks (Moving boxes). Each agent works independently but cohesively. To assign tasks, a "relaxed problem" approach was used utilizing Depth-First Search (DFS) and Breadth-First Search (BFS) algorithms for exploring options, and A* heuristics, with dynamic width assignment, to prioritize paths based on practicality.
    A TLS protected client-server architecture using JRMP communication protocol. A centralized memory-based proxy server acting as an intermediary between the client and the server hoding a TLS protected shared registry. Classpath repositories are shared on a proxy level. JRMP provides access to an authentication service and printer services implemented on the server side. The client is a Spring Boot application with API end-points serving a HTTPS protected User interface.
    DICOMHawk is a honeypot that simulates a real-world DICOM server with a high interaction level. DICOM is an international imaging and communication protocol for medical devices' communication. The honeypot embeds a Web API that allows a user to browse through DICOM image storage. On request, the DICOM server provides a real looking output utilizing "The Cancer Imaging Archive" API with real DICOM files as output. The honeypot embeds a set of honeytokens that allow us to identify the attacker and understand the nature of the attack. Using Elastic stack, the honeypot supports logging and analysis of real-time data.
